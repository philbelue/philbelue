What We Built

A single-page web app deployed on GitHub Pages

The Command Center is a self-contained HTML application — no login, no software install, no dependencies. Reps open the URL on their phone or tablet, select their name, and start working their lead list. Every interaction (Visit, Skip, Maybe, conversation logs, contact info) syncs to a shared Google Sheet within 3 seconds.

  • Cards View — Browse all 66 leads with ICP scores, WNC estimates, territory owners, and booth numbers. Filter by status, type (OEM/SI), or hall. Search by company name.
  • Route Planner — Reps build an optimized walking route through the hall, drag-reorder stops, and mark booths visited as they go.
  • Summary Dashboard — Real-time team progress: leads visited, conversion rates, interest breakdown, and WNC projections by territory. This is Brandon's view.
  • Export — One-click CSV export of all lead data with full conversation notes for CRM import post-show.

Lead Qualification Methodology

1,087 exhibitors reduced to 66 qualified targets

Brandon's initial Apollo.io search filtered the full MODEX exhibitor list (1,087 companies) down to 108 qualified leads across OEM and SI categories. We then applied additional validation criteria to arrive at 66 targets:

  • US manufacturing presence confirmed — Not just US sales offices. IXrouters need to be purchased and commissioned domestically, so we validated actual manufacturing or machine building operations in the US/Canada.
  • ICP alignment scored 1–5 — Based on company size, machine complexity (PLC-controlled), automation profile, and fit with IXON's remote access + monitoring use cases.
  • Duplicate and conflict check — Cross-referenced against Salesforce and Apollo to flag existing customers (AWL Automation), active campaigns (NuMove, BEUMER Group), and accounts already being worked.
  • Supply chain focus acknowledged — MODEX is supply chain / material handling oriented, so a smaller qualified list is expected vs. broader automation shows like Automate or PACK EXPO.
WNC Prediction Model

Estimating pipeline value before the show starts

Every lead carries a WNC (Weighted Net Contribution) point estimate with a confidence level. This isn't a quota — it's a pre-show prediction designed to improve with actual post-show results.

WNC Tier Point Range What It Means Typical Profile
Tier 1 0.3 pts 1–24 devices, min 3 IXrouters Small OEM, 50–100 employees, niche machines
Tier 2 1 pt 25–74 devices, min 5 IXrouters Mid-size OEM or active SI, 100–500 employees
Tier 3 3 pts 75+ devices, min 10 IXrouters Large OEM or enterprise SI, 500+ employees

Model Inputs (v1)

  • Employee count — proxy for fleet size and machine volume
  • Revenue — validates scale and purchasing capacity
  • OEM vs. SI — OEMs have more predictable, recurring device deployments; SIs are project-based
  • Machine complexity — PLC-controlled machines are higher value than simple conveyors
  • US/CA manufacturing confirmed — Must have domestic manufacturing presence
  • Install base signals — Existing fleet size, multi-site operations, service contracts
The Compounding Advantage

Every show makes the next one smarter

This isn't a one-show tool — it's a system that improves with every trade show we attend. MODEX is the training data. IPPE, PACK EXPO, Automate, and every future show benefits from the actuals we capture here.

1

Pre-Show: Predict

Load qualified leads with ICP scores and WNC estimates. Assign territories. Set expected value per rep.

2

At-Show: Capture

Reps log visits, contacts, interest level, and conversation notes. Qualitative signals like "machines commissioned overseas" are captured in notes.

3

Post-Show: Measure

Update actual WNC results per lead. Compare predicted vs. actual. Identify which model inputs were most predictive and which disqualifiers to screen earlier.

4

Next Show: Improve

Feed actuals back into the model. Adjust confidence weights. Better predictions → better territory allocation → better headcount and travel budget justification.

What This Enables for Leadership

  • Headcount justification — Data-backed evidence for how many reps to send per show based on predicted WNC yield
  • Show ROI — Compare travel/hotel/time costs against actual pipeline generated, per show and per rep
  • Show selection — Over time, identify which shows produce the highest WNC per rep-day and prioritize accordingly
  • Rep performance — Conversion rates, leads visited, pipeline generated — all tracked and comparable across shows
  • Disqualifier patterns — Learn which signals (e.g., "machines commissioned overseas," "no PLC machines") should eliminate leads earlier in qualification
Technical Architecture

Simple, reliable, zero-cost infrastructure

  • Frontend — Single-file HTML app (~116KB), vanilla JavaScript, no framework dependencies. Hosted on GitHub Pages (free, always-on).
  • Backend — Google Apps Script deployed as a web app. Handles read/write to a Google Sheet. Runs on Google's servers — no PC needs to be on.
  • Data sync — Auto-push every 3 seconds on any change. Pull on app open. Merge strategy: most recent timestamp wins.
  • Offline resilience — Local browser storage (localStorage) keeps data if connectivity drops. Syncs when back online.
  • Access — Restricted to IXON B.V. domain (Google Workspace). Any IXON email can access the sync layer.
